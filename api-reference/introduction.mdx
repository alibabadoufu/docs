## Authentication

All API endpoints require authentication using session tracking and optional user identification.

<Warning>
API endpoints use session-based tracking. Include `session_id` and `track_id` parameters in your requests for proper request tracing and analytics.
</Warning>

### Authentication Headers

```json
{
  "Content-Type": "application/json",
  "X-Session-ID": "your-session-id",
  "X-Track-ID": "your-track-id"
}
```

## Response Format

All API responses follow a consistent structure:

<ResponseExample>
```json Success Response

  "status": "success",
```

</ResponseExample>

<ResponseExample>
```json Error Response

  "error": "Error description",
  "execution_time": 0.456
}
```
</ResponseExample>

## Rate Limits

<Info>
Current rate limits are enforced per session ID:
- **Pipeline Generation**: 10 requests per minute
- **Agent Operations**: 50 requests per minute  
- **Chatbot**: 30 requests per minute
- **Other endpoints**: 100 requests per minute
</Info>

## Overview

Use this endpoint to receive real-time token-by-token updates and tool activity from the chatbot. The stream follows an Anthropic-style event model with well-defined event types for robust clients and UI rendering.

<Info>
Streaming uses Server‑Sent Events (SSE) over HTTP. Set `Accept: text/event-stream` and keep the connection open while consuming events.
</Info>

## Endpoint

```http
POST /api/v1/chatbot/messages/stream
Accept: text/event-stream
Content-Type: application/json
X-Session-ID: <your-session-id>
```

## Event model

The stream emits these events in order, zero or more times where indicated:

- **message_start**: Start of the assistant message
- **ping** (0..n): Heartbeat every ~2 seconds if no other data
- **content_block_start**: Start of a streamable content block
- **content_block_delta** (0..n): Incremental updates for text/tool/preview/usage/interrupt
- **content_block_stop**: End of the current content block
- **message_stop**: End of the assistant message
- **error**: Error payload if something went wrong

## content_block_delta schemas

<Info>
All deltas are incremental. Treat values as partial updates until you receive `content_block_stop`. Platforms use `twitter` (we normalize any `x` to `twitter`).
</Info>

### text_delta (For main chat UI)

Represents incremental assistant text tokens.

| Field | Type | Description | Example |
| --- | --- | --- | --- |
| `event` | string | Event name | `content_block_delta` |
| `data.type` | string | Constant | `content_block_delta` |
| `data.delta.type` | string | Delta discriminator | `text_delta` |
| `data.delta.text` | string | Next chunk of assistant text | `"Absolutely! Here's a..."` |

### tool_delta

Represents incremental tool activity text for a specific action.

| Field | Type | Description | Example |
| --- | --- | --- | --- |
| `event` | string | Event name | `content_block_delta` |
| `data.type` | string | Constant | `content_block_delta` |
| `data.delta.type` | string | Delta discriminator | `tool_delta` |
| `data.delta.tool.action` | string | Tool action (see table below) | `post_content_generation` |
| `data.delta.tool.platform` | string | Target platform. `""` for non‑platform actions | `twitter` |
| `data.delta.tool.text` | string | Incremental tool output text | `"Drafting caption..."` |

Supported payload for `tool_delta`:

| Action | Platform | Payload fields | Description |
| --- | --- | --- | --- |
| `post_content_generation` | `facebook`/`instagram`/`linkedin`/`twitter` | `text` | Caption/content generation stream |
| `post_content_editing` | `facebook`/`instagram`/`linkedin`/`twitter` | `text` | Edited caption/content stream |
| `post_hashtags_generation` | `facebook`/`instagram`/`linkedin`/`twitter` | `text` | Hashtag generation stream |
| `post_hashtags_editing` | `facebook`/`instagram`/`linkedin`/`twitter` | `text` | Hashtag editing stream |
| `generate_post_image` | `""` | `text` | Image generation instruction stream |
| `edit_post_image` | `""` | `text` | Image edit instruction stream |
| `web_crawler` | `""` | `text` | Crawl instruction/status stream |

### preview_delta

Represents incremental UI preview data produced by tools. Fields vary by action.

Common envelope:

| Field | Type | Description |
| --- | --- | --- |
| `event` | string | `content_block_delta` |
| `data.type` | string | `content_block_delta` |
| `data.delta.type` | string | `preview_delta` |
| `data.delta.preview.platform` | string | `Target platform` |
| `data.delta.preview.action` | string | Tool action (see table below) |

Supported payload for `preview_delta`:

| Action | Platform | Fields | Types | Description |
| --- | --- | --- | --- | --- |
| `post_content_generation` | `facebook`/`instagram`/`linkedin`/`twitter` | `post_content` | string | Incremental caption/content preview |
| `post_content_editing` | `facebook`/`instagram`/`linkedin`/`twitter` | `post_content` | string | Incremental edited caption preview |
| `post_hashtags_generation` | `facebook`/`instagram`/`linkedin`/`twitter` | `post_hashtags` | string | Incremental hashtag preview |
| `post_hashtags_editing` | `facebook`/`instagram`/`linkedin`/`twitter` | `post_hashtags` | string | Incremental edited hashtag preview |
| `generate_post_image` | `""` | `post_image_url` | string (URL) | Generated image URL preview |
| `edit_post_image` | `""` | `post_image_url` | string (URL) | Edited image URL preview |

<Tip>
Clients should render previews incrementally and treat them as non-final until `content_block_stop`.
</Tip>

### usage_delta

Represents token usage metrics for either the agent or a specific tool/platform.

| Field | Type | Description | Example |
| --- | --- | --- | --- |
| `event` | string | `content_block_delta` |  |
| `data.type` | string | `content_block_delta` |  |
| `data.delta.type` | string | `usage_delta` |  |
| `data.delta.token_usage.name` | string | Logical name of the unit | `agent` |
| `data.delta.token_usage.action` | string | `agent`, `image_generation`, `image_editing`, or content/hashtag actions | `agent` |
| `data.delta.token_usage.platform` | string | Platform or empty string | `twitter` |
| `data.delta.token_usage.model` | string | Model identifier | `claude-3-sonnet` |
| `data.delta.token_usage.prompt_tokens` | number | Prompt tokens | `120` |
| `data.delta.token_usage.completion_tokens` | number | Completion tokens | `45` |
| `data.delta.token_usage.total_tokens` | number | Sum of prompt + completion | `165` |
| `data.delta.token_usage.epoch` | number | Unix epoch seconds | `1724691200` |

### interrupt_delta

Represents an interruption that requires user confirmation before continuing (e.g., image generation/edit).

| Field | Type | Description | Example |
| --- | --- | --- | --- |
| `event` | string | `content_block_delta` |  |
| `data.type` | string | `content_block_delta` |  |
| `data.delta.type` | string | `interrupt_delta` |  |
| `data.delta.interrupt.name` | string | Interrupt identifier | `await_image_confirmation` |
| `data.delta.interrupt.message` | string | Prompt to display to the user | `"Do you want me to proceed with image generation/editing?"` |

## Request body

<ParamField path="message" type="object" required>
User message to process. Supports text, URL, image, or quote.

<Expandable title="Message fields">
  <ParamField path="message.message" type="string" required>
  The text content to process
  </ParamField>
  <ParamField path="message.attachments[]" type="array">
  Optional list of attachments; items must include `type` and `url`
  </ParamField>
  <ParamField path="message.metadata.resume" type="string">
  Confirmation token to resume an interrupted run
  </ParamField>
</Expandable>
</ParamField>

<ParamField path="mode" type="string" default="prompt2post">
Processing mode, e.g., `prompt2post`, `image2post`, `url2post`, `quote2post`.
</ParamField>

<ParamField path="thread_id" type="string">
Conversation thread identifier for context continuity.
</ParamField>

<ParamField path="user_background" type="object">
User preferences and context (industry, tone, audience, platforms).
</ParamField>

<ParamField path="is_dummy_mode" type="boolean" default>
Enable dummy responses for testing.
</ParamField>

## JavaScript streaming client

<CodeGroup>
```javascript Node.js (Fetch streaming)
import fetch from 'node-fetch';

async function streamChat({ baseUrl, sessionId, body }) {
  const res = await fetch(`${baseUrl}/chatbot/messages/stream`, {
    method: 'POST',
    headers: {
      'Accept': 'text/event-stream',
      'Content-Type': 'application/json',
      'X-Session-ID': sessionId,
    },
    body: JSON.stringify(body),
  });

  if (!res.ok || !res.body) throw new Error(`HTTP ${res.status}`);

  const decoder = new TextDecoder();
  for await (const chunk of res.body) {
    const text = decoder.decode(chunk, { stream: true });
    for (const line of text.split(/\r?\n/)) {
      if (!line.startsWith('event:') && !line.startsWith('data:')) continue;
      // Parse SSE frames: pair up event and subsequent data lines as needed
      console.log(line);
    }
  }
}

await streamChat({
  baseUrl: 'https://postreachai-ekc4e7fke7bacehe.southeastasia-01.azurewebsites.net/api/v1',
  sessionId: 'session_123',
  body: {
    message: { message: 'Create a concise LinkedIn post on remote work benefits' },
    mode: 'prompt2post',
    thread_id: 'thread_abc123'
  }
});
```

```javascript Browser (ReadableStream)
async function streamChatBrowser() {
  const res = await fetch('/api/v1/chatbot/messages/stream', {
    method: 'POST',
    headers: {
      'Accept': 'text/event-stream',
      'Content-Type': 'application/json',
      'X-Session-ID': window.sessionId,
    },
    body: JSON.stringify({
      message: { message: 'Create a Twitter thread about sustainable tech' },
      mode: 'prompt2post'
    }),
  });

  const reader = res.body.getReader();
  const decoder = new TextDecoder();
  let buf = '';
  while (true) {
    const { value, done } = await reader.read();
    if (done) break;
    buf += decoder.decode(value, { stream: true });
    let idx;
    while ((idx = buf.indexOf('\n\n')) !== -1) {
      const frame = buf.slice(0, idx);
      buf = buf.slice(idx + 2);
      // Frame may include multiple lines like: event: <type>\ndata: <json>
      console.log(frame);
    }
  }
}
```
</CodeGroup>

## Interrupts and resume

If the agent asks for confirmation (e.g., image generation/edit), you will receive an interrupt delta. To continue, call the non-streaming or streaming endpoint with the same `thread_id` and set `message.metadata.resume` to your confirmation string (e.g., "yes").

```json
{
  "message": { "message": "yes", "metadata": { "resume": "yes" } },
  "thread_id": "thread_abc123",
  "mode": "prompt2post"
}
```

## Troubleshooting

<AccordionGroup>
<Accordion title="I only receive pings, no content">
- Ensure your request body is valid JSON and includes a `message.message` string
- Verify `Accept: text/event-stream` and `Content-Type: application/json`
</Accordion>

<Accordion title="Connection closes unexpectedly">
- Check client/network timeouts. SSE requires a long-lived HTTP connection.
- Do not buffer the entire response in proxies; disable response buffering.
</Accordion>

<Accordion title="I need CORS or reverse proxy support">
- Configure your proxy to allow long-lived responses and disable compression if needed.
- If using Mintlify playground proxy, see `api.playground.proxy` settings.
</Accordion>
</AccordionGroup>
